# API Gateway с агрегацией данных и кэшированием

API Gateway (или Backend for Frontend — BFF) объединяет данные из нескольких микросервисов и предоставляет клиентам единое REST API. Сервис поддерживает агрегацию ответов, кэширование данных и
маршрутизацию запросов.

## Функциональность
| Функциональность | Статус |
|------------------|----------|
| Retry & Fallback (Polly) | Сделано |
| Rate Limiting | Сделано |
| Реальные микросервисы (HTTP клиенты) | Сделано |

## Архитектура
**Компоненты системы:**
- **API Gateway** (.NET 8): Единая точка входа, агрегация данных, маршрутизация
- **Сервис пользователей**: Управление профилями пользователей
- **Сервис товаров**: Каталог товаров и информация о продуктах
- **Сервис заказов**: Управление заказами пользователей
- **Redis**: Распределенное кэширование агрегированных ответов
- **Seq**: Централизованное хранение и анализ структурированных логов

## Быстрый старт

### Предварительные требования
- [Docker](https://www.docker.com/get-started) и Docker Compose
- [.NET 8 SDK](https://dotnet.microsoft.com/download) (для локальной разработки)

### Запуск проекта

### Клонирование репозитория

git clone https://github.com/Layvu/api-gateway-bff.git

cd api-gateway-bff

### Запуск всех сервисов через Docker Compose
docker-compose up --build -d
